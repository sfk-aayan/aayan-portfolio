## Architecture

Single-page React/TypeScript portfolio deployed to GitHub Pages at base path `/aayan-portfolio/`. One scrollable page styled as a six-sheet engineering drawing set ("Issued for Construction" concept): a dark cyanotype-blueprint hero (Sheet 01) pinned under a light paper print (Sheets 02–06) that slides over it on scroll.

**Content lives in `src/data.ts`** — all text, projects, experience, skills, publications, and certifications are exported constants; `src/types.ts` defines the interfaces. Content is fixed: design work must not alter `data.ts`. The `id` fields there (`SYS-01`, `EXP-01`, `CERT-01`…) are legacy from a previous theme and are used only as React keys — on-screen reference codes are generated by the components (`FIG. 01`, `DETAIL A`, `S-01`, `C-01`, `T-01`).

**Spatial structure in `src/App.tsx`:**
- A `sticky top-0 h-[100svh]` wrapper pins `Hero` (Sheet 01, blueprint) at z-0, with a scroll-linked dim overlay.
- The paper stack (`ProjectsSection` = Sheets 02–05, `PhilosophySection` = Sheet 06, footer colophon) sits at z-10 with `bg-paper` and a deep top shadow, sliding over the hero.
- Fixed chrome: left scroll ruler (z-30, in App.tsx), title block (z-40) and nav chip (z-50) in `Navbar`, `SheetTransfer` overlay (z-8000), `CustomCursor` (z-9997+).
- The root div uses `overflow-x-clip`, NOT `overflow-x-hidden` — hidden would break the sticky hero.

**App startup:** `BootSequence` (pen-plotter calibration animation) runs first, then calls `onComplete` to reveal content via opacity transition; `CustomCursor` and `SheetTransfer` mount only after boot.

**Sheet-number contracts** (must stay in sync when adding/reordering sections):
- `useActiveSection` watches section ids: `section-hero`, `section-bio`, `section-experience`, `section-projects`, `section-research`, `section-philosophy`.
- `Navbar` maps those sections to sheet numbers/labels (`PLATES`, `SHEET_OF`).
- `GlitchHeader` receives `node="SHEET NN · LABEL"` and regex-extracts the numeral for the ghost margin number.

**Navigation:** nav links don't scroll — `requestSheetTransfer()` (in `SheetTransfer.tsx`) dispatches a custom event; the overlay wipes across, jump-scrolls instantly while covered, then wipes off. `section-hero` is special-cased to `scrollTo(0)` because the sticky hero's rect is always at viewport top. Honors `prefers-reduced-motion` with a plain instant jump.

**Component naming caveat:** `GlitchHeader` (plate header) and `ScanReveal` (plot/clip-path reveal) keep their historical names and prop signatures from the old theme; their `color` props ("amber"/"cyan"/"emerald") are remapped internally to safety-orange/drafting-blue/ink.

**`CinematicVideo`** contains deliberate streaming/preload logic (fetch → progress → blob URL, fade-on-loop). Preserve it when restyling; the cyanotype look comes from `grayscale` + `mix-blend-luminosity` over the solid blueprint background.

**Animation library is `motion` v12 — import from `"motion/react"`, never `"framer-motion"`** (only present as a transitive dep). Shared easing/duration constants live in `src/lib/motionConfig.ts`.

Legacy hooks `useTypewriter`, `useScrambleText`, and `useIsDesktop` exist but are currently unused.

## Styling

Tailwind CSS v4 via the `@tailwindcss/vite` plugin (not PostCSS). All design tokens are `@theme` variables in `src/index.css`:
- Paper world: `paper`, `vellum`, `ink`, `ink-soft`, `ink-faint`, `rule`
- Blueprint world: `print`, `print-deep`, `chalk`, `chalk-soft`, `chalk-faint`
- Working accents: `safety` (international orange), `draft` (drafting blue), `stamp` (rubber-stamp red)

Fonts (Google Fonts import): Fraunces (display serif), Inter (sans), IBM Plex Mono (annotations).

Custom utilities in `src/index.css`: `.millimeter` / `.millimeter-chalk` (graph grids), `.paper-grain`, `.stamp`, `.ghost-numeral(-chalk)` (outline numerals), `.caret`, `.tick-travel`, `.protractor-spin`, plus the `plot-sweep` keyframes used by `ScanReveal`.

The system cursor is hidden only under `@media (pointer: fine)`; `CustomCursor` draws a full-viewport crosshair using white + `mix-blend-difference` so it reads over both the blueprint and paper zones (same trick as the scroll ruler).

## Static Assets

Public assets (video, favicon) go in `public/` and must be referenced via `import.meta.env.BASE_URL` prefix (not hardcoded paths) to work with the GitHub Pages base path — see `VIDEO_URL` in `Hero.tsx`. The favicon (`public/images/favicon.svg`) is referenced relatively from `index.html`.
